= Analyser les événements FSx pour ONTAP EMS dans NetApp Workload Factory
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
Identifiez et résolvez rapidement les problèmes du système de fichiers FSx for ONTAP avec l'analyseur d'événements intelligent dans NetApp Workload Factory. L'analyseur d'événements extrait et analyse automatiquement les événements du système de gestion des événements (EMS) FSx for ONTAP, en tirant parti de l'IA Agentic avec l'intégration Amazon Bedrock.



== Description de la tâche

Les administrateurs de stockage réagissent souvent aux événements FSx for ONTAP EMS uniquement après des réclamations de clients, ou en maintenant des scripts et des alarmes personnalisés. Cette approche réactive peut réduire l'efficacité, retarder la résolution des problèmes et augmenter l'interruption.

L'analyseur d'événements extrait automatiquement les événements EMS d'erreur, d'avertissement et d'urgence de FSx for ONTAP file systems. Vous pouvez consulter ces événements en link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["se connecter au système de fichiers à l'aide d'un lien"] et en link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["octroi des autorisations de _consultation, de planification et d'analyse_"^] dans votre compte AWS. Les événements sont affichés pendant 72 heures avant d'être supprimés.

Grâce à l'intégration d'Amazon Bedrock, Workload Factory utilise l'IA pour analyser les événements et fournir des informations exploitables afin de maintenir la santé et les performances de vos systèmes de fichiers FSx for ONTAP.

Les principaux avantages sont les suivants :

* Dépannage avancé : l’IA identifie, analyse et fournit automatiquement des informations pour corriger les événements FSx for ONTAP EMS, réduisant ainsi le temps d’investigation manuelle.
* Solution optimale : L’analyseur d’événements fournit des étapes claires et concrètes pour résoudre les événements FSx for ONTAP EMS.


Lorsque vous utilisez l'analyseur d'événements, vous gardez le contrôle total de votre environnement tout en bénéficiant d'une analyse IA avancée.

Pour permettre à Workload Factory d'analyser les événements, vous devez activer Amazon Bedrock, sélectionner le modèle que Workload Factory utilise, créer un point de terminaison privé pour se connecter à Amazon Bedrock, ajouter des autorisations et créer une licence d'entreprise.

link:https://aws.amazon.com/bedrock/pricing/["Tarifs d'Amazon Bedrock"^]



=== confidentialité et sécurité des données

Votre confidentialité des données et votre sécurité sont protégées par :

* *Souveraineté des données* : Toutes les données et agrégations restent au sein de votre compte AWS et sont communiquées via un point de terminaison VPC privé (Amazon Bedrock), sans aucune exposition sur Internet public.
* *Aucun entraînement d'IA* : les données client ne sont pas utilisées pour entraîner ou améliorer les modèles. Amazon Bedrock traite les événements en temps réel mais n'effectue aucun entraînement sur vos données. Les résultats sont stockés uniquement dans votre environnement.


Pour plus de détails, veuillez vous référer au link:https://docs.aws.amazon.com/bedrock/latest/userguide/data-protection.html["Documentation sur la protection des données d'Amazon Bedrock"^].



== Avant de commencer

Pour utiliser l'analyseur d'événements, assurez-vous des points suivants :

* Vous avez link:https://docs.netapp.com/us-en/workload-setup-admin/add-credentials.html["autorisations _opérations et remédiation_"^] dans votre compte AWS pour analyser les événements des systèmes de fichiers FSx for ONTAP.
* Le port 22 (SSH) est ouvert dans le groupe de sécurité associé à votre système de fichiers FSx for ONTAP.


Exigences supplémentaires (le système vous le demandera lors de l'analyse des erreurs de journalisation):

* *Modèle Amazon Bedrock*
+
Configurez les API Amazon Bedrock pour chaque compte AWS. Les API Amazon BedRock sont utilisées pour fournir des informations sur les événements FSx for ONTAP.

+
Modèle recommandé : `anthropic.claude-sonnet-4-20250514-v1:0`. Veuillez fournir l’ARN du profil d’inférence pour la région sélectionnée.

* *Lien vers Workload Factory*
+
Créez et associez un lien à un système de fichiers FSx for ONTAP pour activer l'analyse des événements basée sur l'IA. Un lien établit une relation de confiance entre Workload Factory et un ou plusieurs systèmes de fichiers FSx for ONTAP et exploite AWS Lambda.

+
link:https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["Apprenez à associer un lien existant ou à créer et associer un nouveau lien"]. Une fois le lien associé, revenez à cette opération.

* *Autorisations AWS IAM*
+
Ajoutez les autorisations suivantes à la stratégie associée au rôle IAM de NetApp Workload Factory.

+
** `bedrock:InvokeModel`
** `bedrock:InvokeModelWithResponseStream`
+
Ces autorisations permettent à Workload Factory d'utiliser les modèles Bedrock pour l'analyse des erreurs et la fourniture de recommandations de correction. Ce profil garantit également un accès sécurisé à l'IA pour des analyses personnalisées.

+
Ajoutez également les autorisations suivantes pour les informations d'identification AWS associées à Workload Factory :

** `bedrock:GetInferenceProfile`
** `bedrock:ListInferenceProfiles`
+
Ces autorisations vérifient la disponibilité du modèle.







== Visualisez et analysez les événements EMS pour FSx for ONTAP

Utilisez la console Workload Factory pour visualiser et analyser les événements EMS pour les systèmes de fichiers FSx for ONTAP .

.Étapes
. Connectez-vous à l'aide de l'un des link:https://docs.netapp.com/us-en/workload-setup-admin/console-experiences.html["expériences de la console"^].
. Sélectionnez le menu image:workloads-menu-icon.png["L'icône du menu hamburger permet d'accéder aux charges de travail telles que le stockage, l'analyse de données d'entreprise (EDA), l'intelligence artificielle (IA), les bases de données, VMware et l'administration."] puis sélectionnez *Stockage*.
. Dans le menu Stockage, sélectionnez *Analyse*.
. À partir de l'écran d'analyse, sélectionnez les comptes AWS, les informations d'identification et les régions contenant les systèmes de fichiers FSx pour ONTAP que vous souhaitez analyser.
+
Seuls les systèmes de fichiers FSx pour ONTAP avec affichage des événements à l'écran.

. Si nécessaire, complétez la configuration de l'analyse IA en suivant les instructions à l'écran pour satisfaire aux prérequis manquants.
. Trouvez le système de fichiers FSx pour ONTAP que vous souhaitez analyser, puis sélectionnez *Afficher les événements*.
. Consultez les informations détaillées sur l'événement.
